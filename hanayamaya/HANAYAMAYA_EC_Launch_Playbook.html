Task 01 / Phase 0

ドメイン確定・取得

ブランドの「住所」を決める。これが全ての起点。

クライアント

hanayamaya.com をムームードメインで取得

事業計画では hanayamaya.com を想定。年額¥1,408。ムームードメインなら即日取得可能。

ここで考えるべきことが3つある。

① .com vs .jp の判断。 .comは国際展開を見据える場合に強い。ただし、高級米という日本ローカル商材の場合、hanayamaya.jp のほうが信頼感が高い可能性がある。.jpは年額約¥3,000。コスト差は微少なので、ブランドイメージで判断すべき。事業計画の「Reclaim the center.」という英語タグラインと黒基調の世界観からすると、.comのほうがグローバル感があり合致する。

② サブドメイン設計。 現時点では hanayamaya.com（LP）+ shop.hanayamaya.com（Square）が最もシンプル。メールは info@hanayamaya.com。将来的にブログや定期便専用ページが必要になった場合もサブドメインで対応可能。

③ 取得タイミング。 ドメインは早い者勝ち。「hanayamaya」は一般的な単語ではないため取られるリスクは低いが、事業計画が固まったなら今日取得しても問題ない。DNS反映に最大48時間かかるため、他のインフラ構築に先行して取るべき。

意思決定が必要

hanayamaya.com で確定するか、.jp を検討するか。推奨は .com（事業計画の世界観との整合性）。

具体的な手順

  • ムームードメイン（muumuu-domain.com）にアクセス
  • 「hanayamaya.com」を検索 → 取得可能か確認
  • アカウント作成（クライアントの個人名義 or 法人名義）
  • Whois情報公開代行を「有効」に設定（個人情報非公開のため）
  • 支払い方法を設定（クレジットカード推奨・自動更新ON）
  • 取得完了後、DNS設定はEC支援チームに引き継ぎ

注意点

ドメインの自動更新は必ずONにする。更新を忘れてドメインが失効すると、第三者に取られるリスクがある。ブランドドメインの喪失は致命的。

Task 02 / Phase 0

Squareアカウント開設・本人確認

決済インフラ。審査に数営業日かかるため、最優先で着手。

クライアント

Square for Retailを開設し、本人確認審査を通す

Squareを選んだ理由は明確：初期費用ゼロ・月額固定費ゼロ・決済手数料3.6%のみ。初期段階で月額費用が発生するBASEやShopify（月額$39〜）は不要。売上が月50万を超えるまではSquareが最適。

審査で必要なもの：

準備物

  • メールアドレス — Google Workspace設定前なので、まずは個人メールでOK。後から法人メールに変更可能
  • 本人確認書類 — 運転免許証 or マイナンバーカード
  • 銀行口座情報 — 売上入金先。個人口座でもOK（個人事業主として登録する場合）
  • 事業情報 — 事業形態（個人事業主 or 法人）、業種（食品小売）、年間売上予測

ここで考えるべきこと：個人事業主 vs 法人。

テスト販売フェーズ（4月〜9月）は個人事業主で始めるのが合理的。法人設立は売上が安定してからでよい。個人事業主でSquareを使う場合、屋号「HANAYAMAYA」で登録できる。インボイス制度への対応は別途freeeで管理する。

Squareの審査は通常1〜3営業日だが、食品販売の場合に追加確認が入る可能性がある。最悪1週間を見込んでおく。だからこそ最優先で着手が必要。

意思決定が必要

① 個人事業主として登録するか、法人を設立してから登録するか（推奨：個人事業主で即開始）
② 入金サイクルの確認（Squareは翌営業日入金。ただし銀行によっては翌週）

落とし穴

Squareの「オンラインストア」機能と「決済リンク」機能は別物。HANAYAMAYAの場合、LPからの導線は決済リンク（Square Checkout Link）を使う。Squareのオンラインストアに独自LP機能があるが、今回は使わない（自前のLP + Netlifyのほうがデザイン自由度が高い）。ただし商品管理はSquareダッシュボードで行う。

Task 04 / Phase 0

配送業者の選定

米2kgの配送。コスト・品質・オペレーション負荷のバランス。

クライアント

ヤマト運輸 vs 日本郵便 vs 佐川急便を比較し、確定する

米2kgの配送は60サイズ・2kg以内で収まる。これは全キャリアの最小サイズ帯。

配送業者サービス60サイズ料金（関東→関東）特徴
ヤマト運輸宅急便¥940〜追跡◎ 時間指定◎ ブランド印象◎
日本郵便ゆうパック¥870〜コンビニ持込割引あり
佐川急便飛脚宅配便¥770〜法人契約で最安
日本郵便レターパックプラス¥600（全国一律）2kg・A4サイズ 厚さ制限なし

ここで深く考えるべきこと。

① 送料は「原価」ではなく「体験」の一部。 事業計画では送料を¥900と想定。ヤマト宅急便でほぼ原価。だが、ここでケチって佐川にすると「高級ブランドなのに佐川」という印象リスクがある。ヤマトの宅急便は消費者の信頼度が最も高い。¥3,800の高級米にふさわしい配送体験を提供するならヤマト一択。

② 法人契約で送料を下げる。 ヤマトは月間出荷数に応じて法人割引がある。ただし、テスト販売段階（月10〜30個）ではまだ交渉力がない。まずは個人契約（ヤマトビジネスメンバーズ or クロネコメンバーズ）で始め、月間50個を超えたら法人契約を交渉。

③ 配送時の温度管理。 米は常温配送で問題ないが、真夏（7〜8月）は高温多湿による品質劣化リスクがある。ただし2kgなら消費速度が速い（約2週間分）ため、通常配送でOK。「届いたら冷暗所で保管してください」の一言を同梱物に入れる。

④ 発送元の住所。 花山からの発送か、別拠点からの発送か。花山から直送できるなら「産地直送感」が出る。ただし毎日の発送オペレーションが花山で回るかは要確認。

意思決定が必要

① 配送業者：ヤマト宅急便で確定するか
② 発送拠点：花山から直送か、別拠点から発送か
③ 時間帯指定：購入時に選べるようにするか（推奨：Squareの注文フォームにノート欄を設けて対応）

Task 05 / Phase 0

梱包仕様の最終決定

箱を開けた瞬間が、ブランド体験のクライマックス。

クライアント＋チーム

「開封体験」をブランド体験の一部として設計する

Amazonの段ボールを開ける体験と、HANAYAMAYAの箱を開ける体験は、別物でなければならない。¥3,800を払った顧客が箱を開けた瞬間に「これは正しい選択だった」と思える演出が必要。

梱包の3層設計：

第1層：外箱（段ボール）

60サイズのクラフト段ボール。外側はシンプルでOK。ただしテープの色は黒にする（茶色のOPPテープではなく）。これだけで印象が変わる。できれば箱に小さなHANAYAMAYAロゴのスタンプを押す。コストは最小限で「ここはこだわっている」と伝わる。

第2層：薄紙（ティッシュペーパー）

箱を開けたとき、商品がむき出しではなく黒い薄紙で包まれている。この「一枚の紙」が開封体験を劇的に変える。黒の薄紙は100枚¥1,000程度。1枚あたり¥10のコストで高級感が3倍になる。

第3層：商品パッケージ（黒マット袋）

すでにデザイン済みの黒基調パッケージ。これがブランドの「顔」。薄紙を開けた瞬間、この黒い袋が現れる。それだけで「特別なものを手にした」感覚になる。

黒テープ¥51巻¥300 / 約60回分
黒薄紙¥10100枚¥1,000

事業計画では梱包費を¥250と見積もっていた。実際は¥140〜180で収まるため、¥70〜110の余裕がある。この余裕分で同梱物（花山の景色カードなど）を充実させられる。

Task 06 / Phase 1

特定商取引法に基づく表記

EC販売に法的に必須。これがないと違法。

クライアント＋チーム

LPに特商法表記ページを追加する

特定商取引法に基づく表記は、EC販売において法的義務。表記がないまま販売を開始すると行政処分の対象になる。

必須記載事項：

記載が必要な項目

  • 販売業者名 — 個人事業主の場合は本名（屋号だけでは不可）
  • 代表者名
  • 所在地 — 省略不可。ただし個人事業主の場合、請求があれば遅滞なく開示する旨の記載で住所の省略が可能（2022年法改正）
  • 電話番号 — 同上。請求時開示でも可
  • メールアドレス — info@hanayamaya.com
  • 商品の販売価格 — ¥3,800（税込）
  • 送料 — 送料込み
  • 支払方法 — クレジットカード（Square経由）
  • 支払時期 — 注文時に決済
  • 商品の引き渡し時期 — 注文から◯営業日以内に発送
  • 返品・交換について — 食品のため原則不可。不良品の場合のみ交換

意思決定が必要

① 住所・電話番号を公開するか、請求時開示とするか（推奨：プライバシー保護の観点から請求時開示。ただし「請求があれば遅滞なく開示します」の文言は必須）
② 商品の引き渡し時期：注文から何営業日以内に発送するか（推奨：3〜5営業日）

よくある失敗

「所在地：宮城県」のような都道府県のみの記載はNG。省略する場合は必ず「請求があれば遅滞なく開示します」の文言を入れる。この文言がないと法的要件を満たさない。

Task 07 / Phase 1

プライバシーポリシー策定

個人情報保護法対応。顧客の信頼基盤。

ECチーム

個人情報の取り扱いに関する方針を策定しLPに掲載

GA4でアクセス解析をする時点で、Cookie（個人情報）を取得している。プライバシーポリシーなしでのGA4利用は個人情報保護法に抵触するリスクがある。

記載すべき内容：

必須項目

  • 収集する個人情報の種類（氏名、住所、メール、電話番号、決済情報、アクセスログ）
  • 利用目的（商品の発送、注文確認、マーケティング、サービス改善）
  • 第三者提供の有無（Square：決済処理、ヤマト運輸：配送、Google：アクセス解析）
  • 個人情報の管理方法
  • 開示・訂正・削除の請求方法
  • Cookieの使用について（GA4）
  • 問い合わせ窓口

テンプレートはネット上に多数あるが、HANAYAMAYAのトーンに合わせて丁寧に書く。法律文書だからといって無味乾燥にする必要はない。「お客様の大切な情報を、責任を持ってお預かりします」という姿勢が伝わる文面に。

Task 08 / Phase 1

食品表示ラベル作成

食品表示法。これを間違えると回収騒ぎになる。

クライアント＋チーム

食品表示法に準拠した表示ラベルを設計する

精米の食品表示はJAS法に基づく玄米及び精米品質表示基準で定められている。米の場合、一般的な加工食品とは表示ルールが異なる。

必須表示項目

  • 名称：精米
  • 原料玄米：産地（宮城県）、品種（ひとめぼれ）、産年（令和7年産）、使用割合（単一原料米 or ブレンドの場合は割合）
  • 内容量：2kg
  • 精米時期：精米年月日 or 精米時期（◯年◯月上旬/中旬/下旬）
  • 販売者：氏名（法人名）+ 住所 + 電話番号

ここで深く考えるべきこと。

① 「産年」の表示。 2026年4月にローンチする場合、販売するのは令和7年（2025年）産の米。「令和7年産」と明記する必要がある。新米の時期（10月〜）には「令和8年産 新米」と切り替わる。ラベルの印刷を大量に刷ると産年切り替え時に在庫が無駄になる。ラベルは小ロットで印刷するか、産年部分だけシールで対応するのが現実的。

② 精米時期の鮮度管理。 米は精米後から酸化が始まる。精米時期の表示は消費者の購買判断に直結する。理想は注文を受けてから精米する「受注精米」。これが実現できれば「注文後に精米してお届け」という最強の鮮度訴求ができる。ただしオペレーション負荷は上がる。テスト販売段階では、まとめて精米して在庫する方式でも問題ない。

③ 「単一原料米」の表示。 HANAYAMAYAは花山産ひとめぼれ100%なので「単一原料米」と表記できる。これは品質の証明になる。ブレンド米は割合表示が必要になり、ブランド価値が下がる。

法的リスク

食品表示の虚偽・誇大表示は食品表示法違反で罰則あり（個人：懲役2年以下 or 罰金200万円以下）。特に産地偽装は社会的信用を完全に失う。表示内容は100%正確でなければならない。確認は必ず複数人で行う。

意思決定が必要

① 受注精米にするか、在庫精米にするか（推奨：テスト段階は在庫精米。軌道に乗ったら受注精米へ移行）
② ラベル印刷方法：自社プリント（ラベルシール）vs 印刷会社発注

Task 09 / Phase 1

返品・キャンセルポリシー

食品ECの返品は「原則不可」。だが、書き方が信頼を左右する。

ECチーム

食品ECとして適切な返品ポリシーを策定する

食品は衛生上の理由からお客様都合の返品は受け付けないのが業界標準。ただし「返品不可」とだけ書くと冷たい印象になる。HANAYAMAYAのブランドトーンに合わせて、誠実さが伝わる表現にする。

ポリシー骨子

  • お客様都合の返品：食品の特性上、ご容赦いただいております
  • 商品不良・配送事故の場合：到着後7日以内にご連絡いただければ、交換または全額返金で対応
  • キャンセル：発送前であれば対応可能。発送後のキャンセルは返品扱い
  • 定期便の解約：次回発送日の7日前までにメールで連絡。解約手数料なし

重要：不良品対応は「即座に・気持ちよく」。 高級ブランドの不良品対応は、ブランドの真価が問われる瞬間。配送事故で袋が破損していた場合、面倒な手続きを要求するのではなく、写真1枚メールで送ってもらうだけで即交換。この体験が口コミを生む。

Task 10 / Phase 1

利用規約

万が一のトラブルに備える法的な盾。

ECチーム

EC販売の利用規約を策定する

特商法表記やプライバシーポリシーとは別に、サービス利用の条件を定めたもの。テスト販売段階では簡潔なもので十分。

盛り込むべき内容

  • 適用範囲（本サイトでの商品購入に適用）

  • 準拠法・管轄裁判所

Task 11 / Phase 2

DNSは「ドメインをどのサーバーに向けるか」を設定するもの。HANAYAMAYAの場合、1つのドメインで2つのサービスを動かす：

DNS設定内容

  • Aレコード / CNAMEレコード：hanayamaya.com → Netlify（LP表示用）
  • MXレコード：hanayamaya.com → Google Workspace（メール用）

事業計画のKPIは「LPのCVR 1.5%以上」。これを計測するにはGA4が必須。

計測すべきイベント

  • ページビュー：LPの各セクションのスクロール到達率（10%, 25%, 50%, 75%, 90%）

Task 36 / Phase 6

Square定期便（サブスク）機能実装

事業モデルの核心。「LTVの最大化」はここにかかっている。

ECチーム

Squareのサブスクリプション機能で定期便を構築する

事業計画：「LTV粗利49,800円（毎月配送・2年継続）」。この数字を実現するのがサブスク機能。

定期便の設計

  • 配送頻度：毎月 / 隔月の2択を用意
  • 価格：単発と同じ¥3,800 or 定期便割引（¥3,600等）→ 割引するかは要検討
  • 初回特典：花山の季節の手紙 + 特別同梱物（定期便だけの体験）
  • 解約条件：縛りなし。次回発送7日前までに連絡。解約手数料ゼロ
  • 休止機能：旅行や一時的に不要な場合の「1回スキップ」を許可

定期便の「割引」はするべきか？

議論が分かれるところ。割引すると「安くなるから定期便にする」というコスト動機になる。HANAYAMAYAが目指す「心理的依存（このお米が切れると生活のリズムが乱れる）」とは相性が悪い。推奨は同額で、定期便限定の「特別体験」（季節の手紙・花山の景色カード）で差別化する。 値引きではなく体験の付加価値で定期便への転換を促す。

実装時期は8月。 ソフトローンチから3ヶ月間の単発販売データ（リピート率）を見てから定期便の設計を最終確定する。
